## 2026-07-02 M0-foundation [GOOD]

- 事象: T1〜T6の全impl委譲が一発で全ゲート（install/test/lint/build/prettier）通過し、出口一括opusレビューもPASS（C0/H0）
- 原因: 計画v2.2のタスク粒度（成果物ファイル・依存・完了条件が明記済み）と、impl委譲プロンプトに「完了条件コマンド列＋exit code報告」を義務付けた運用
- 一般化ルール (次ループの指示文としてそのまま使える形で): impl委譲プロンプトには必ず「仕様の正の§番号」「完了条件のコマンド列とexit code報告義務」「スコープ外ファイルの明示」を含める
- 反映先: loop prompt
- 発生回数: 1 回目

## 2026-07-02 git初期化事故（セッション操作） [BAD]

- 事象: フォルダに既存の.git（ユーザーの手元コミット）があるのに新規初期化前提でgit操作し、checkoutで作業ツリーからファイルが一時的に消えた（既存コミットが全ファイルを保持していたため完全復旧、損失ゼロ）
- 原因: セッション開始時の環境情報「git repoではない」を、時間経過後（ユーザーが手元でinit/commit済み）も再検証せず使った
- 一般化ルール (次ループの指示文としてそのまま使える形で): 状態を変えるgit操作（init / checkout / branch -D / reset）の直前に必ず `git status && git log --oneline -3` で現状を確認する。セッション開始時のスナップショット情報は陳腐化する前提で扱う
- 反映先: CLAUDE.md 昇格候補
- 発生回数: 1 回目
