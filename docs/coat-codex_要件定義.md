# coat-codex 要件定義（Fable計画フェーズ申し送り用）

## 概要
ミニチュア・プラモデル・ガンプラ等を問わない、塗装レシピ（工程記録）の作成・保存・出力ツール。
DBを持たないクライアントサイド完結型。GitHubプロジェクト名: `coat-codex`

---

## 1. アーキテクチャ
- **サーバー側DBなし**。全データはブラウザ内保存（localStorage / IndexedDB）
- 写真もIndexedDBに保存（Cookieはサイズ制限があるため実装上はIndexedDB/localStorageを使用）
- デプロイ: **Cloudflare Pages**（静的 + クライアントサイドSPA）で完結。Workers / D1 / R2 は原則不要
- フロントエンド: **React推奨**（サイトのコーデックス/秘伝書系デザインとの親和性、Cloudflare Pagesでの実績を考慮）※最終決定はOpus/Sonnet実装フェーズに委任可
- データ消失は自己責任（利用規約に明記する想定）

## 2. データモデル
- レシピ = 対象物（ミニチュア／プラモデル等）1体分の塗装記録
- レシピ単位で「使用カラー一覧」「使用ツール一覧（筆／スポンジ／エアブラシ等）」を先に定義
- **ベース工程**（全体プライマー／全体スプレー等、特定パーツに紐付かない工程）と、**部位（パーツ）ごとの工程**の2種類を持つ。ベース工程は全体写真に、パーツ工程は各パーツに紐づく
- 部位（パーツ）ごとに、上記一覧から色・ツールを参照し、工程（順序・技法メモ）を記録
- 塗料指定方法（2方式併用）:
  1. 主要ブランド（Citadel / Vallejo / AK等）プリセット色DBから選択
  2. プリセットにない塗料は自由入力＋HEXコード／カラーピッカーで色見本を手動指定
- 写真: パーツ単位で添付可、IndexedDB保存、印刷／PDF／SNS画像に反映

## 3. 出力機能
- 印刷最適化レイアウト
- PDFダウンロード
- SNS投稿支援: 写真4枚以下を自動生成し、各SNSの投稿フォームを起動。対象は **X（旧Twitter）／Bluesky**。将来的にMastodon等を追加しやすいよう、投稿先を拡張可能な設計にしておく
- **note.com向けMarkdown出力**（note.com投稿を想定したフォーマットでのエクスポート）

## 4. データ管理
- JSONファイルでのエクスポート／インポート（バックアップ・端末移行用。初期入力画面からインポートしてレシピ編集を再開できる）
- **素のMarkdownエクスポート**（JSONエクスポートと隣り合わせで配置。note.com向け出力とは別に、汎用的な人が読める文書としての保存・共有用）

## 5. スコープ
- 対象: ミニチュア全般＋プラモデル／ガンプラ等を含む広範囲の塗装記録ツール（特定ゲームシステムに限定しない）
- コンテンツ: ツール単体（記事／解説コンテンツなし）※方針変更あれば別途申し送り

## 6. 言語対応
- 日本語＋英語（切り替え式、i18n対応）

## 7. デバイス対応
- レスポンシブ必須、PC／スマホ均等優先

## 8. デザイン
- サイト名「coat-codex」に沿った"秘伝書"テイスト
- ビジュアルデザインはClaude Designで別途検討（Fable計画では機能要件のみでよい）

## 9. 名称・命名候補（既存類似サービスとの重複回避済み）
- 採用: `coat-codex`
- 予備候補: `pigment-ledger` / `layer-log` / `brush-formula` / `swatch-forge`
- 確認済みの類似先行サービス（名称衝突回避のため参考）: Paint Pad (paintpad.app), PaintMyMinis (paintmyminis.de), PaintVault (paint-vault.com), 'Eavy Archive (eavy-archive.com), Miniature Painter Pro

## 10. 画面構成（手書きワイヤーフレームより）
※ユーザーが手書きでラフに書いた入力項目案。画面遷移・入力項目の意図を言語化したもので、レイアウト自体の最終形ではない。**より使いやすいUI/UX構成があれば、実装フェーズ（Opus/Sonnet）側で積極的に代案を提案してよい。**

### 10-1. 初期入力画面
- タイトル入力
- 全体写真アップロード
- 使用カラー登録（メーカー＋カラー名。この画面で全部確定させなくてもよく、後から追加可能な想定）
- 使用ツール登録（後述のパーツ登録画面で、ここに登録した内容から選択できるようにする）
- **JSONインポート**（過去にエクスポートしたレシピを読み込んで編集再開できるようにする。新規作成とインポートの2導線をこの画面に用意）
- 「make codex!」ボタンで次画面（ペイント構成全体表示）へ遷移

### 10-2. ペイント構成全体表示画面（レシピ全体のサマリー）
- 工程（1st paint, 2nd paint...）をカード形式でリスト表示し、ドラッグ&ドロップで並び替え可能
- 各カードをタップ/クリックすると「パーツごとのペイントレシピ登録」画面へ遷移
- **ベース工程**（全体プライマー／全体スプレー等、特定パーツに紐付かない工程）にも対応。表示位置は初期入力画面で登録した全体写真の上（オーバーレイ）でよい
- 混合色の割合を数値で表示（例: 「25% + 75%」のような表示例あり。詳細ロジックは10-3参照）
- パーツ写真があればサムネイル表示
- 画面下部に出力アクション: 印刷／X投稿／Bluesky投稿／note.com向けMarkdown出力（投稿用）に加えて、**JSONエクスポートと素のMarkdownエクスポートを隣り合わせで配置**（バックアップ・閲覧用途、3・4章参照）。将来的にMastodon等のSNS追加も見据えた拡張可能な設計とする

### 10-3. パーツごとのペイントレシピ登録画面
- パーツ名入力
- パーツ写真添付（+ボタンで追加、複数枚想定）
- 工程（#工程1, #工程2...）を+ボタンで複数追加可能。各工程は下記を1セットとして持つ:
  - 技法名（一般的な技法名をセレクト or 自由入力。例: ベースコート／レイヤー／ウォッシュ／ドライブラシ等）
  - 使用塗料をA, B, C...と+ボタンで追加（**最大5種混合まで対応**）
    - 各塗料: メーカー（セレクト or 入力）→ カラー名（セレクト or 入力、選択したメーカーで候補を絞り込み）
    - 色見本プレビュー（メーカー・カラー名から自動表示。プリセットにない場合はカラーチップ写真の添付や手動指定にも対応）
  - 複数塗料使用時の混合割合は **%入力／A:B比率入力の両対応**（相互排他ではなく以下のロジックで連動）:
    - %を直接入力した場合、その時点でA:B比率入力欄の内容はクリアする
    - A:B比率で入力した場合、%を自動計算して表示する（例: 3:2 → 60% / 40%）
    - A:B比率入力時は、入力した比率表記そのものも保持しておき、「ペイント構成全体表示」画面（10-2）の混合割合表示に%と併記する（例: 「60% (3:2)」）
  - 工程ごとのフリーメモ欄
