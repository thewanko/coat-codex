# coat-codex — Claude Design向け依頼プロンプト

> 本書はcoat-codexのビジュアルデザイン検討（Claude Design）用資料の一部です。3点セット: [依頼プロンプト](coat-codex_ClaudeDesign_プロンプト.md)／[DesignSystem初期値](coat-codex_DesignSystem.md)／[画面テンプレート](coat-codex_画面テンプレート.md)。機能仕様の原典は [技術計画v2](../coat-codex_技術計画_v2.md)。

## A-1. プロジェクト概要

**coat-codex** は、ミニチュア・プラモデル・ガンプラ等の塗装レシピ（工程記録）を作成・保存・出力するクライアントサイド完結型SPAである。サーバーDBを持たず全データ（写真含む）をブラウザのIndexedDBに保存し、印刷／PDF／SNS投稿支援（X・Bluesky）／note.com向けMarkdown／JSON・Markdownエクスポートの各出力を備える。日英2言語対応、Cloudflare Pagesにデプロイ。実装は React 19 + TypeScript + Vite、テーマは CSS カスタムプロパティ（`src/styles/theme.css`）で一元管理する前提のため、**デザイントークンはケバブケースのCSS変数として返却してほしい**。

## A-2. ターゲットユーザー

- ミニチュアペインター（ウォーハンマー等）／プラモデル・ガンプラモデラー。特定ゲームシステムに限定しない広い層
- 「自分の調色・工程を後で再現したい」「SNSやnoteでレシピを共有したい」人
- 塗装ブースの脇でスマホ片手に記録する場面と、PCでじっくり整理する場面の両方がある（**PC/スマホ均等優先**）
- 日本語圏・英語圏の両方

## A-3. デザインテーマ「秘伝書（codex）」の方向性

### 雰囲気キーワード
- 錬金術師の調合帳／中世写本（illuminated manuscript）／工房（アトリエ）の記録簿
- 羊皮紙と墨、活版印刷の罫線、封蝋（シーリングワックス）、金箔のアクセント、製本・栞
- 「図鑑・標本ラベル」的な整然さ — 秘伝書だが**読み物ではなく実用の台帳**である

### 参考にすべき質感・モチーフ
- ごく淡い紙の質感（テキスト背後ではノイズを抑える）、レタープレス風の細罫・角飾り
- 封蝋レッド・アンティークゴールドを「押印」「章題」的にポイント使用
- 「make codex!」ボタン＝封蝋を押す・製本する、といった儀式感のあるマイクロコピー/演出の余地あり
- 見出しはセリフ体（和文明朝系＋欧文Garamond系）、本文・UIはサンセリフで実用性を担保するハイブリッド

### 避けるべき表現
- **色見本（スウォッチ）に影響する装飾**: スウォッチや写真にセピア調・カラーフィルタ・色付きオーバーレイを乗せることは絶対不可。色の正確な見えがこのツールの生命線
- テキスト背後の強いテクスチャノイズ・低コントラスト（羊皮紙色の上でもWCAG AAを維持）
- ブラックレター（髭文字）の本文使用、和文の斜体（明朝イタリックは崩れる）
- 2010年代スキュオモーフィズムの革表紙・金縁キッチュ、ダークファンタジー一辺倒（ガンプラ層も使う）
- 装飾のための装飾でタップ領域・情報密度を犠牲にすること

## A-4. 全画面リストと必須UI要素

全7ルート＋共通シェル＋1ダイアログ。**[固定]＝機能上動かせない要素・挙動、[自由]＝デザイン裁量あり**。詳細な構造ラフは同梱の「ドキュメントC 画面テンプレート」を参照。

### 0. AppShell（全画面共通）
- [固定] `LanguageSwitcher`（ja/en切替）／`AppFooter`（`/terms` へのリンク常設）／`ToastHost`（保存・エラー通知）
- [自由] ヘッダー/フッターの造形、ロゴ表現、トーストの見た目

### 1. HomePage `/`
- [固定] レシピ一覧（updatedAt降順）、カードごとの未バックアップドット、カードメニュー（開く／複製／削除／JSONエクスポート）、新規作成ボタン、JSONインポートボタン、`StorageStatusBar`（保護状態バッジ・使用量・最終エクスポート）、`ExportReminderBanner`（条件表示・「あとで」スヌーズ）
- [自由] グリッド/リスト等の一覧形式、カードの装丁（「蔵書」「巻物」等の見立ては歓迎）、空状態の演出

### 2. RecipeSetupPage `/recipe/:id/setup`（10-1 初期入力）
- [固定] タイトル入力／全体写真アップローダ（複数可・**先頭＝代表写真**・並び替えで代表変更）／使用カラー登録（PaletteEditor）／使用ツール登録（ToolListEditor）— **各行に使用数バッジ「N工程で使用中」を表示し、工程から参照中の色・ツールは削除ボタン無効化＋理由表示（機能上動かせない）**／**JSONインポート導線を新規作成フォームと並置**／「make codex!」ボタン
- [自由] フォームの流れの演出（章立て・段階表示等）、「make codex!」の儀式的表現

### 3. RecipeOverviewPage `/recipe/:id`（10-2 全体表示）
- [固定] 代表写真＋`BaseStepOverlay`（ベース工程名チップ列を写真上にオーバーレイ、タップ/編集ボタンで `/recipe/:id/part/base` へ。未登録時は「＋ベース工程を追加」）／`OverviewPhotoStrip`（2枚目以降）／`PartCardList`（**パーツカードのD&D並び替え**。サムネ・工程数・混合バッジ「60% + 40% (3:2)」、タップで編集へ）／パーツ追加ボタン／`ExportActionBar`（印刷・**PDFダウンロード（明示ボタン必須）**・X・Bluesky・note MD・**JSONと素Markdownは隣接配置**）／`ExportReminderBanner` コンパクト帯
- [自由] オーバーレイの意匠、カードのドラッグ中表現、アクションバーのグルーピング（ただしJSON/素MD隣接は維持）

### 4. PartEditorPage `/recipe/:id/part/:partId`（10-3）＋ baseモード `/recipe/:id/part/base`
- [固定] モバイル＝フルページ／**PC幅＝Overview上のスライドインパネル**（同一コンポーネント）。通常: パーツ名入力＋写真ギャラリー（＋ボタン複数追加）。baseモード: 固定見出し「ベース工程（全体）」＋代表写真の読み取り専用サムネ（写真編集はSetupへ誘導）。`StepList`（**工程D&D並び替え＋上下移動ボタン併設**）／`StepCard`（技法セレクトor自由入力・塗料スロット最大5＝A〜E・ブランド→カラー絞り込み・スウォッチチップ・**%↔A:B連動の混合比入力**・ツール選択・メモ・工程削除）／工程追加ボタン
- [自由] StepCardの情報レイアウト、A〜Eのラベル表現、パネルの開閉演出

### 5. PrintViewPage `/recipe/:id/print`
- [固定] 印刷最適化レイアウト（レシピ全文＋**色見本が主役**）、`PrintToolbar`（印刷実行・PDF保存案内、`@media print` で非表示）
- [自由] 紙面デザイン全般 — ここが「秘伝書」らしさの見せ場。ただしA-5の印刷制約に従うこと

### 6. TermsPage `/terms`
- [固定] 利用規約・**データ消失自己責任の免責明記**・ブラウザストレージの性質説明（i18n静的コンテンツ）
- [自由] 文書ページの組版

### 7. ShareDialog（Overviewから起動するモーダル）
- [固定] 合成画像プレビュー（**最大4枚**・1枚目＝代表写真ベース）／投稿テキストエディタ＋ターゲット別カウンタ（X＝重み付き280字、Bluesky＝300 grapheme。超過警告＋自動トリムボタン）／A系統: 「共有シートで投稿（画像付き）」主ボタン＋失敗時B系統フォールバック＋「うまく共有できない場合」常設リンク／B系統: 手順ガイド①DL②投稿画面③手動添付＋「合成画像をダウンロード」＋Intent起動ボタン＋**画像手動添付の案内常時表示**
- [自由] プレビューの見せ方、手順ガイドの図解

## A-5. 制約（全デザイン共通）

1. **レスポンシブ**: PC/スマホ均等優先。基準ビューポート mobile 390px / PC 1280px、ブレークポイント768px（PCスライドインパネルの分岐点）。タッチターゲット44px以上
2. **日英2言語**: 英語ラベルは日本語の1.5〜2倍長になり得る（例:「複製」→ "Duplicate"）。ボタン・バッジ・タブは可変幅前提、固定幅ラベル禁止。和文本文は line-height 1.7 目安
3. **印刷**: 色見本が主役 — スウォッチは十分な面積＋**カラー名・hex・混合比テキストを必ず併記**（モノクロ印刷でも情報が残る）。背景テクスチャ・ベタ塗りは印刷では抑制（インクコスト・裏写り）。パーツ単位の改ページ制御を想定
4. **アクセシビリティ**: WCAG 2.1 AA（本文4.5:1、大文字・UI部品3:1）。羊皮紙背景上でも墨色テキストでコントラスト確保。色だけに依存しない（スウォッチには常に名前＋hex）。フォーカスリングを羊皮紙上で視認可能に。D&Dには上下ボタン代替あり（既定仕様）
5. **スウォッチ中立性**: `SwatchChip` の周囲は白ベースの中立フレームとし、テーマ色の影響を受けない領域として設計する

4. **コンポーネントスタイル仕様**: ドキュメントBのコンポーネントインベントリ準拠。各バリアント＋hover/focus/active/disabled/dragging状態
5. **print.css の方向性**: PrintViewPageの紙面サンプル1枚（A4想定）
6. **装飾アセット方針**: 罫線・角飾り・封蝋・章題オーナメント等をSVGで供給する場合の一覧と使用ルール
